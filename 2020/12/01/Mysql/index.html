2 SQL

2.1 SQL 的基本概念

1 什么是SQL

structured Query Language,结构化查询语言，其实就是定义了所有操作关系型数据库的规则。

2 SQL通用的语法

  • SQL以单行或多行书写，以分号结尾
  • Mysql数据库的SQL查询语句不区分大小写，关键字建议用大写
  • 3种注释的方式
    • 单行注释：– 注释内容或者 # 注释内容
    • 多行注释：/* 注释 */

3 SQL的分类

01 DDL（Data Definition Language）数据定义语言，操作数据库，表。关键字：creat, drop, alter等

02 DML（Data Manipulation Language）数据操作语言，用来对数据库中表中的数据进行增删改，关键字：insert, delete, update等

03 DQL（Data Query Language）数据查询语言，用来查询表中的记录。关键字：select , where等

04 DCL（Data Control Language）数据控制语言，用来定义数据库的访问权限，安全级别，及创建用户，关键字：GRANT, REVOKE

2.2 DDL：操作数据库，表

1 操作数据库 CRUD

  • Create 创建

    • 创建数据库

      • create database 数据库名称

    • 创建数据库，判断是否存在

      • create database if not exists 数据库名称；

    • 创建数据库并指定字符集名称

      • create database 数据库名称 character set 字符集名

    • 例子：创建db4的数据库，判断是否存在，并且指明字符集

      • create database if not exists db4 character set gbk;

  • Retrieve查询

    • 查询所有数据库的名称

      • show databases；

    • 查看某个数据库字符集，即数据库的创建语句

      • show create database 数据库名称；

  • Update 修改

    • 修改数据库的字符集
      • alter database 数据库名称 character set 字符集名称；

  • Delete 删除

    • 删除数据库

      • drop database 数据库名称；

    • 删除数据库，判断是否存在

      • drop database if exists 数据库名称；

  • 使用数据库

    • 查询当前正在使用的数据库名称

      • select database( );

      • 使用数据库，进入到文件夹里面

        • use 数据库名称

2 操作表 CRUD

  • create 创建

    • 1 语法

      • create table 表名（列名1 数据类型1，列名2 数据类型2，……）;

    • 2 数据类型

      • 01 int:整数类型

        • age int;

      • 02 double: 小数类型(小数最多有几位，小数点后面保留几位)

        • score double(5,2)

      • 03 date: 日期类型 ，只包含年月日，yyyy-MM-dd

      • 04 datetime: 日期时间，包含年月日时分秒，yyyy-MM-dd HH:mm:ss

      • 05 timestamp:时间戳类型

        • 将来不给这个字段赋值或者不给这个字段赋值为NULL,默认使用当前系统时间，来自动赋值

      • 06 varchar 字符串类型

      • name varchar(字符串最大长度);

      • zhangsan 8个字符，张三 2个字符

  • 创建表例子

    • create table student ( id int, name varchar(32), age int , score double(4,1),birthday date, insert_time timestamp);使用大括号更加清晰

    • 复制表

      • create table like 被复制的表名

  • retrieve 查询

    • 查询某个数据库中所有表的名称
      • show tables;
    • 查询表结构
      • desc(description)表名称；

  • update 修改表

    • 01 修改表名

    • alter table 表名 rename to 新的表名；

    • 02 修改表的字符集

      • alter table 表名称 character set 字符集名称

      • alter table stu character set gbk;

      • 查看表的字符集

        • show create table 表名；

    • 03 添加一列

      • alter table 表名 add 列名 数据类型；

    • 04 修改列的名称 类型

      • alter table 表名 change 列名 新列名 新数据类型;

      • alter table 表名 modify 列名 新数据类型;

    • 05 删除列

      • alter table 表名 drop 列名；

  • delete 删除表

    • drop table 表名；
      • drop table if exists 表名；

2.3 DML：增删改表中的数据

  • １添加数据

    • 语法

      • insert into　表名（列名1，列名2，列名3……）values(值1，值2，值3……)

    • 注意

      • 01 列名和值要一一对应

      • 02 如果表名后不定义列名，则默认给所有列添加值

      • 03 除了数字类型，其他类型要用引号引起来，单双引号都可以

  • ２删除数据

    • 语法

      • delete from 表名 [where 条件]；当条件满足的时候就从这张表里面删除数据

    • 注意

      • 如果不写条件，就会删除表里面的所有数据，需要一条一条去删

      • 如果要删除表里所有记录

        • 01 delete from 表名；——不推荐使用，有多少条记录就会执行多少次删除操作

        • 02 truncate table 表名; 删除表建立一个一模一样的空表，推荐使用，效率更高

  • ３修改数据

    • 语法

      • update 表名 set 列名 1=值1，列名2=值2 where 条件

    • 注意

      • 如果不加任何条件，则会将表中的所有记录全部修改

==6. 分组查询==

语法
group by 分组字段
注意
01 分组之后查询的字段，要么是分组字段，要么是聚合函数，此时分完组，一个组是一个整体，不要出现个体，否则就会失去分组的意义

02 where 和having 的区别

  • where 在分组之前进行限定，如果不满足这个条件，则不参与分组

​ having 在分组之后限定，如果不满足结果，则不会被查询出来

  • where 后不可以跟聚合函数，Having后可以跟聚合函数
